# AI法規制モニタリングサイト 要件定義書

## 1. プロジェクト概要

### 目的
各国のAI法規制を定期的にモニタリングし、更新情報をブログ形式で蓄積・公開するWebサイトを構築する。

### コンセプト
- シンプルで運用しやすい
- バックエンド不要
- HTMLベースの静的サイト
- Googleスプレッドシートで監視対象を管理

---

## 2. システム構成

### 2.1 データ管理

#### Googleスプレッドシート（監視対象管理）
最小限の情報のみ保持：

| 列 | 項目名 | 説明 | 必須 |
|----|--------|------|------|
| A | 国名 | 例：EU, 米国, 日本 | ○ |
| B | 調査対象URL | 監視するサイトのURL | ○ |
| C | 通知用メールアドレス | 更新時の通知先（任意） | - |

**例：**
```
国名    | 調査対象URL                           | 通知用メール
--------|---------------------------------------|------------------
EU      | https://ec.europa.eu/ai-act          | you@example.com
米国    | https://www.whitehouse.gov/ai        | you@example.com
日本    | https://www8.cao.go.jp/cstp/ai/      |
```

#### HTMLファイル（記事蓄積）
- `index.html` に記事をブログカード形式で蓄積
- 新しい記事が上部に追加される
- 静的ファイルとして永続化

---

## 3. 機能要件

### 3.1 監視対象管理
- Googleスプレッドシートで直接編集
- 行の追加：新しい監視対象を追加
- 行の削除：監視対象を削除
- 行の編集：URL、メールアドレスの変更

### 3.2 サイトチェック機能
カスタムコマンド `/check-site` で実行：

1. Googleスプレッドシートから監視対象リストを取得
2. 各URLにアクセスして更新を確認
3. 更新があった場合：
   - AI法規制の概要を抽出
   - 重要な本文の一部を引用
   - 元ソースへのリンクを含める
4. ブログカード形式で記事を生成
5. `index.html` に追記
6. （メールアドレスがあれば）更新通知を送信

### 3.3 記事表示機能
- ブログカード形式で一覧表示
- 最新の記事が上部に表示
- 国別のタグ表示
- レスポンシブデザイン

---

## 4. ブログカード構成

各記事カードに含まれる情報：

| 項目 | 説明 |
|------|------|
| 国名タグ | EU, 米国, 日本など |
| 日付 | 記事追加日 |
| タイトル | 法規制の名称や更新内容 |
| 概要 | 更新内容の要約（2-3文） |
| 本文抜粋 | 重要な条文や説明の引用 |
| ソースリンク | 元サイトへのリンク |

## 6. 運用フロー

### 6.1 初回セットアップ
1. Googleスプレッドシートを作成
2. 監視対象URLを入力
3. スプレッドシートを公開設定（読み取り専用）
4. プロジェクトにスプレッドシートIDを設定

### 6.2 日常運用
```
あなた「/check-site を実行」
     ↓
Claude「スプレッドシートから監視対象を取得」
     ↓
Claude「各URLをチェック」
     ↓
Claude「更新があればブログカード生成」
     ↓
Claude「index.htmlに追記」
     ↓
Claude「（任意）メール通知送信」
     ↓
あなた「index.htmlを開いて確認」
```

### 6.3 監視対象の追加・削除
1. Googleスプレッドシートをブラウザで開く
2. 行を追加・削除・編集
3. 保存（自動保存）
4. 次回の `/check-site` から反映される

---

## 7. 非機能要件

### 7.1 パフォーマンス
- ページ読み込み：2秒以内
- 軽量なHTML/CSS（画像最小限）

### 7.2 互換性
- モダンブラウザ対応（Chrome, Firefox, Safari, Edge）
- モバイル対応（レスポンシブデザイン）

### 7.3 保守性
- シンプルな構成で誰でもメンテナンス可能
- バックエンド不要
- サーバー不要

### 7.4 拡張性
- 将来的に国別フィルター機能を追加可能
- RSSフィード生成も検討可能

## 9. カスタムコマンド仕様

### `/check-site`
**機能**: 全監視対象サイトをチェックし、更新があれば記事を生成

**処理フロー**:
1. Googleスプレッドシート読み込み
2. 各行のURLをチェック
3. 更新検知（前回との差分）
4. 記事生成（タイトル、概要、本文抜粋、リンク）
5. HTMLに追記
6. メール通知（設定されている場合）

**出力**:
- チェック結果のサマリー
- 追加された記事の件数
- エラーがあればその詳細

## 11. 制約事項

- リアルタイム更新ではない（手動でコマンド実行が必要）
- サイト構造が変わると抽出ロジックの調整が必要
- Googleスプレッドシートの公開設定が必要
